# PRODUCT — auteur

## One line

**auteur is a Claude Code skill that directs a website like a film** — art
direction is locked on a commit-sheet before a line of code, assets are
generated by local CLIs, and every ship is gated by an *executable* anti-slop
linter and a real motion check.

## Who it's for

Designers and engineers who drive Claude Code and want **award-tier
interactive sites** — WebGL heroes, scroll cinema, generative motion — instead
of the template output most AI web tooling produces. People who can tell the
difference between "a nice gradient with text" and a site a skeptical art
director would respect.

## The 10-second test

A skeptic opens the repo. In ten seconds they should be able to:

1. See **four live interactive sites** the skill actually built (not
   screenshots — real WebGL you can drag).
2. Run **one command** (`slopscan`) against any of them and watch it pass, or
   point it at typical AI slop and watch it fail.

If both hold, the tool has earned a second look. Everything in this repo is
built backward from that test.

## What it actually does

The skill hands Claude Code a **director's pipeline**, enforced in this order:

1. **Commit-sheet** — one committed art direction (a single brand hue, a
   typographic system, a motion budget, named anti-references) written down
   *before* any markup. No "let's see how it looks" drift.
2. **Assets** — imagery, video frames, depth maps and 3D geometry generated by
   local CLIs (Codex / Gemini image gen, Blender headless, Depth-Anything),
   routed by cost and by what each tool is actually good at.
3. **Build** — one WebGL context, GSAP/Lenis scroll, DOM motion on
   transform/opacity only, from a small set of proven recipes (fluid sim, GPGPU
   particles, glitch-type, 3D worlds, scroll-morph state machines).
4. **Gate** — nothing ships until it passes:
   - **slopscan** — a zero-dependency, regex-based linter that fails the build
     on concrete slop patterns (see below). Runnable by anyone, in CI.
   - **motionqa** — a Playwright pass that drives the page on a throttled CPU
     and fails on dropped frames, long tasks, autoplay sound, or console
     errors.

## Anti-references (the slop it refuses)

These are hard-coded refusals, not taste preferences. The linter enforces
several of them mechanically:

- The **250–290° purple→blue gradient** that every AI defaults to.
- **SaaS-cream** hero gradients and centered-logo-on-gradient heroes.
- **Inter / Space Grotesk** as the reflexive default typeface.
- `transition: all`, `addEventListener('scroll')` for animation, autoplay
  audio, video with no poster, WebGL with no `prefers-reduced-motion` branch.
- **Hero-metric blocks with invented numbers** ("10x faster", "99.9%") — this
  repo contains none, and neither does anything the skill ships.
- Stock-photo-plus-text-overlay as a "hero".

## Accessibility floor (non-negotiable)

- `prefers-reduced-motion` → a **rich still render**, never a blank page.
- All copy readable with **JavaScript off**.
- No full-frame strobe / epilepsy risk.
- Responsive at 390 / 768 / 1440 with **no horizontal overflow**.

## What this repo does NOT claim

- **No benchmark numbers.** auteur is a design discipline, not a system with a
  throughput figure. There is no fabricated metrics table anywhere in this
  repo. The only quantitative claim is `slopscan` results (0 fails / 0 warns),
  which you can reproduce in one command.
- **No "award-winning."** The four showcase sites are the argument. Judge them
  yourself.
